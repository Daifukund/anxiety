**DESIGN & UX** => no definitive = need to check what is the best
COLORS
Primary background = white purity, breathing space, no visual overload
Accent color = soft violet => calm, serenity, trust
Secondary color = light blue => modern touch, emotional vibe
Emergency color = soft pastel red => stands out clearly for urgent help without being aggressive 
TYPOGRAPHY
Front = Inter => simple, universal, highly readable
Medium for titles + Regular for body text + Bold only for CTAs (buttons)
BUTTONS
Shape = fully rounded (pill style) => friendly and approachable
Normal buttons = light blue or soft violet background with white text
SOS button = red background with white text => the only red element in the app, place prominently 
UX PRINCIPLES
Use lots of white space = lowers cognitive load and anxiety
Keep icons simple (line style, iOS-like)
Use soft animations (fade-in, smooth slide) instead of sharp transitions
NON-FUNCTIONAL REQUIREMENTS 
PERFORMANCE
App must launch in <2 seconds.
SOS flow should be usable immediately even if offline.
Smooth animations (breathing/grounding visuals run at 60fps).
RELIABILITY
Core SOS flow available 100% offline.
Minimal crash tolerance (target <1 crash per 1,000 sessions).
Data sync with backend should be resilient: queue locally if offline, sync later.
SECURITY & PRIVACY
Journaling data stored locally first (encrypted if possible).
No mandatory account creation (privacy-first, optional login for sync).
GDPR-compliant (data deletion upon request).
SCALABILITY
Backend should support up to 10,000 DAUs without performance degradation.
Modular architecture: easy to add new exercises/tools without breaking SOS flow.
COMPATIBILITY
iOS first (iPhone 12 and above, iOS 16+).
Designed for both light and dark mode.
USABILITY 
SOS button must always be visible and accessible in â‰¤2 taps.
Accessibility support: large text mode, haptic/audio feedback for visually impaired users.
